Loading GFF3 Annotation Files

The GenoSets systems allows you to load annotation files in GFF3 format.

File format requirements:

The file must contain the nucleotide FASTA sequences for each of the seqid (column 1) listed in the file. It may contain additional nucleotide sequences that do not have associated feature annotations and these assembled units will be saved in the database. The inclusion of protein sequences in the FASTA section is optional.

Important
If the protein sequences are omitted, then the types of the features to be translated must be specified in the wizard (see below).


The locus tag and product of the feature is determined from the attributes column (column 9) of the file.  The locus tag can be specified by either the attribute tags 'locus_tag' or 'ID'.  The product is set to the attribute 'product'.

Steps:

  1. Download the GFF3 file of interest to local machine.

  2. Open the Load File Wizard.

    • On the Add Data menu, click Load Data Wizard.
    • Select Generic Feature Format Version 3 (GFF3) from the GFF folder.

  3. Select the local file.

  4. Select an existing organism or create a new organism.

    You can add an additional annotation set to an existing organism or you may create a new organism.

  5. Specifying feature types to be translated:

    Important
    If the protein sequences are not included in the FASTA section of the file, then the feature types must be specified in the wizard.  

    The most common type is 'CDS' but in some cases the type is specified as 'gene'. If the type is 'gene' then the file most likely does not contain any 'CDS' features. Note: GenBank/EMBL/DDBJ and PATRIC GFF3 annotations often include both a gene and CDS entry for a single locus. In this case, the translation type should be specified as 'CDS'.  Multiple feature types may be seperated by line, tab, comma, or semicolumn.

  6. Click Next

  7. Enter the details for the annotation method.
